The Foundry

Agentic Feature Pipeline · GitHub Issue → PR

Оркестратор, который берёт GitHub Issue и автоматически проводит его через полный цикл разработки: анализ контекста, планирование, реализацию, верификацию и открытие PR.

Каждая задача исполняется в изолированном git-worktree. Всё состояние хранится в append-only SQLite. Прогресс переживает рестарты процессов.

Pipeline — линейный FSM

01
FETCH
→
gh issue list по SOURCE_REPO + ISSUE_LABELS/ASSIGNEE/MILESTONE. Сортировка по priority/p0 → p1. Upsert в SQLite.
02
CONTEXT
→
Анализирует worktree: языки, манифесты, релевантные файлы (TF-IDF). Читает repo_memory — touched_files, verify_commands, common_failures.
03
PLAN
→
Агент генерирует план. Маркер NEED_VERIFICATION → BLOCKED: комментарий в issue, задача ждёт ответа человека.
04
IMPLEMENT
→
Агент пишет код в изолированном worktree. Checkpoint: git diff → data/checkpoints/. До каждой retry-попытки — git reset --hard HEAD.
05
VERIFY
→
① Детерм.: pytest / ruff / npm test / cargo test — short-circuit при ненулевом exit. ② LLM-ревьюер: анализирует git diff, возвращает PASS / FAIL / UNCLEAR.
06
PR
→
git commit → push foundry/task-{id} → gh pr create. Закрывает issue комментарием со ссылкой на PR.
↻ Retry-цикл (шаги 4–5): до MAX_IMPLEMENT_ATTEMPTS=2 попыток. Каждая следующая получает план + отчёт предыдущей верификации. BLOCKED → возобновляется через POST /api/tasks/{id}/resume.

Стек, хранилище и безопасность

Технологии
Python 3.11+ FastAPI Click CLI React 19 TypeScript Vite SQLite structlog Langfuse (опц.) claude_cli codex_cli opencode_cli Docker Compose uv gh CLI
SQLite — таблицы
tasks статус, стадия, worktree path, branch, pr_url
task_events append-only лог: stage_started/finished, agent_tool
stage_results input/output каждой стадии, per-attempt
repo_memory долгосрочная память: файлы, тест-команды, сбои
agent_sessions session_id claude_cli для resume
Безопасность и изоляция
git worktree — каждая задача в своём worktrees/task-{id}, основная ветка недоступна агенту
Denylist команд — rm -rf, git push --force, git checkout main, git reset --hard вне worktree → исключение
SAFE_AGENT_MODE=true — Claude без --dangerously-skip-permissions, Codex без bypass-sandbox
Env scrubbing — subprocess агента получает только базовый allowlist + ключ конкретного backend
Sanity check PR — отказ commit если >40 файлов или затронуты .venv/ / __pycache__
Checkpoints — git diff до каждой попытки в data/checkpoints/; failed worktree не удаляется
